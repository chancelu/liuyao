# 六爻网站 MVP 信息架构 + 页面线框说明 v0.1

## 一、整体信息架构

### 1.1 站点主结构
- 首页
- 起卦问题页
- 摇卦页
- 排盘生成页
- 结果页
- 注册 / 登录页
- 历史记录页
- 分享结果页

### 1.2 用户主路径
**主路径：** 首页 → 起卦问题页 → 摇卦页 → 排盘生成页 → 结果页 → 注册保存
**注册用户：** 首页 / 历史页 → 再次起卦 → 结果页 → 保存 / 分享 / 回看

### 1.3 导航结构
**顶部导航（简单）：**
- Logo / 品牌名
- 开始起卦
- 历史记录（登录后）
- 登录 / 头像

**页脚（轻量）：**
- 关于六爻（轻）
- 使用说明（轻）
- 隐私 / 条款
- 联系方式（后期）

## 二、页面间跳转逻辑

### 游客路径
1. 首页点击开始起卦 → 起卦问题页
2. 填写完成 → 摇卦页
3. 摇完 6 次 → 排盘生成页
4. 生成完成 → 结果页
5. 点击保存 / 更多体验 → 注册页
6. 注册完成 → 保存成功 → 可进历史页

### 注册用户路径
1. 首页 / 历史页 → 起卦问题页
2. 完整体验 → 结果页
3. 自动保存或提示保存
4. 可从历史页回看

## 三、页面线框说明

### 3.1 首页 Home

**页面目标：** 把用户带进氛围、让用户想试、不让小白害怕

**区块 1：Hero 首屏**
- 大标题 + 副标题 + 主 CTA（开始起卦）+ 次 CTA（看看如何使用）
- 背景：全屏氛围图层
- 用户心理：这是什么？看起来很高级。我可以试一下。

**区块 2：三步说明**
- 3 张横向卡片：写下问题 → 摇卦 → 看排盘与解读
- 用户心理：原来这么简单，我不需要先懂六爻

**区块 3：示例问题引导**
- 标题：你也可以这样问
- 多个标签式示例问题，点击可直接进入起卦页并带入问题
- 用户心理：我知道怎么开口了

**区块 4：轻量内容感**
- 什么是六爻（1 小段）+ 结果会如何展示
- 用户心理：看起来不是纯玄学忽悠，有一定专业感

**区块 5：再次 CTA**
- 开始第一次起卦

**首页不该有的东西：** 过多导航、过长介绍、一堆术语、很重的学习内容、太明显的工具入口排列

### 3.2 起卦问题页 Ask

**页面目标：** 帮助用户把问题说清楚，不至于卡住

- 标题：你想问什么？/ 副说明：问题越具体，结果越清楚
- 大输入框（placeholder 给启发）
- 分类选择（pills / segmented cards）
- 时间范围选择
- 背景补充（选填，多行输入框）
- 示例问题引导（可点击填入）
- CTA：开始摇卦

**用户心理：** 我不知道该怎么问，但这个页面在帮我整理问题

### 3.3 摇卦页 Cast

**页面目标：** 做出整个产品的"品牌记忆点"

- 顶部信息条：问题摘要 + 第 X / 6 次 + 当前进度
- 中央交互主舞台：三枚铜钱视觉 + 摇卦按钮 + 动画反馈 + 每次结果显现
- 结果反馈区：这一次已记录为第 X 爻，可轻显示老阴/老阳/少阴/少阳
- 已完成爻位概览：6 条爻位的小型进度显示
- 操作区：摇这一爻 / 重新开始 / （可选）上一步

**用户心理：** 这个过程很特别，我在参与不是在填表

### 3.4 排盘生成页 Processing

**页面目标：** 把等待变成"推演过程"

- 结果摘要：已生成主卦 / 变卦 + 问题摘要
- 推演状态：按步骤动态出现（正在装盘 → 推演世应与六亲 → 分析用神与旺衰 → 生成本次结果）
- 氛围视觉：轻微动态背景，抽象"展开/推演/显现"效果

**用户心理：** 系统不是在瞎转圈，它正在做一件复杂而认真的事

### 3.5 结果页 Result

**页面目标：** 让用户先获得"信任" → 再获得"结论" → 再看到"解释" → 最后触发"保存/分享/再次起卦"

**区块 1：顶部摘要** — 问题、分类、提问时间

**区块 2：排盘总览（第一屏核心）**
- 本卦、变卦、动爻、世应、简化版六爻盘
- 左侧：卦名/基础信息 | 中间：现代视觉化排盘 | 右侧/下方：关键参数摘要
- **关键要求：必须"像产品设计过的排盘"，不能是开发直接吐数据**

**区块 3：初步结论**
- 一句话判断 + 当前趋势 + 一小段解释
- 示例：结果倾向：当前偏阻，但后续有转机

**区块 4：详细分析**
- 4.1 白话分析（默认展开）：核心是什么、为什么这样、后面怎么走、怎么理解
- 4.2 专业分析（折叠展开）：用神、旺衰、动变、世应、象法、应期
- Tooltip 术语解释

**区块 5：操作区** — 保存结果 > 分享 > 重新起卦

### 3.6 注册 / 登录页 Auth

**页面目标：** 不打断主体验，承接"保存结果"动机

- 文案：保存这次卦例，以后可随时回看
- 邮箱注册表单
- 轻收益说明：保存历史结果、查看专业版内容、继续更多体验次数

**用户心理：** 我不是被迫登录，而是为了保存这次结果

### 3.7 历史记录页 History

**页面目标：** 让产品从一次性体验变成可回来的工具

- 标题：我的卦例
- 卡片列表：问题标题、分类、日期、一句话结论、本卦/变卦简要标签
- 空状态：去开始第一次起卦

**用户心理：** 这里像我的私人记录夹，不是冷冰冰的数据库

### 3.8 分享结果页 Share

- 摘要展示：问题（可匿名）+ 本卦/变卦 + 一句话结论
- 简化说明：本次结果来自在线六爻占卦体验
- CTA：我也来起一卦
