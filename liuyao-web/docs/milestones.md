## 系统架构原则：三层分离

### 1. 确定性层（排盘引擎）
摇卦结果转爻、装盘、排盘、计算参数。不能依赖大模型。

### 2. 推理层（分析引擎）
结合 liuyao-calculation skill 做六爻分析。

### 3. 展示层
用户可读输出、普通版/专业版、历史记录和复盘。

## 技术栈
- 前端：Next.js + React + Tailwind CSS（不用 shadcn/ui）
- 数据与鉴权：Supabase（Auth + Postgres + Storage）
- 部署：Vercel
- AI：服务端统一调用模型，前端不直接碰模型密钥
- i18n：next-intl（预埋，当前只中文）

## 多语言预埋方案
- 第一版只上线中文
- 所有文案进入 locale 文件
- 页面组件不写死中文文案
- 路由结构可扩展 locale 段
- 分享图模板预留 locale 参数

## 并行工作流
- **工作流 A（产品/设计）：** 页面细节确认、视觉规范、高保真设计
- **工作流 B（前端）：** 页面搭建、组件实现、交互动效
- **工作流 C（后端/领域逻辑）：** 数据模型、排盘引擎、analysis service
- **工作流 D（数据/运营）：** 埋点、报表、日志与监控
