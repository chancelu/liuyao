# 六爻网站开发路线图与执行规则

## 1. 开发总原则
- 先做主链路：首页 → 提问 → 摇卦 → 排盘 → 分析 → 结果 → 注册保存
- 先把系统跑通，再做更深视觉 refinement
- 第一版中文上线，但预埋多语言架构
- 排盘与分析严格分层
- 有阶段性进展要主动同步，不等用户追问
- 有卡点、外部依赖、停滞时要立刻反馈，不空转

## 2. 当前方向锚点
后续开发必须遵守以下已确认内容：
- 邮箱注册 / 登录
- 游客先体验，后注册保存
- 游客 1 次，注册后更多
- 高级模式第一版不做
- 中文优先，预埋 i18n
- MVP 必须包含：
  - 示例问题
  - tooltip 术语解释
  - 分享链接
  - 分享图片
  - 重新起卦
- 结果页顺序固定：
  1. 先排盘
  2. 再初步结论
  3. 再具体分析

## 3. 里程碑
### Milestone 0：项目底座
已完成 / 已有骨架：
- Next.js + TypeScript
- Tailwind
- i18n 预埋
- 路由骨架
- 基础页面
- mock 流程
- Supabase client / env 骨架

### Milestone 1：主流程前端闭环
目标：前端可完整跑通。

包括：
- 首页
- 起卦问题页
- 摇卦页
- 生成页
- 结果页
- 登录页
- 历史页

### Milestone 2：排盘引擎
目标：把“摇卦 → 排盘”做成确定性引擎。

包括：
- 主卦 / 变卦生成
- 动爻判断
- 世应
- 纳甲
- 六亲
- 六神
- 月建 / 日辰 / 旬空
- 标准 chart JSON
- 测试样例

### Milestone 3：分析系统
目标：排盘结果进入 skill / 模型分析链路。

包括：
- analysis input schema
- analysis output schema
- 初步结论
- 白话分析
- 专业分析
- 失败记录与排查能力

### Milestone 4：账户 / 保存 / 历史 / 分享
目标：完成“先体验后注册”的闭环。

包括：
- 邮箱注册 / 登录
- 游客 1 次体验限制
- 保存结果
- 历史记录页
- 分享链接
- 分享图

### Milestone 5：体验打磨
目标：把“能用”推到“值得用”。

重点打磨：
- 首页
- 摇卦页
- 结果页
- tooltip
- 动效
- 分享图

### Milestone 6：埋点 / 报表 / 上线准备
目标：可验证、可监控、可上线。

包括：
- 埋点
- 漏斗报表
- 分类分布
- 错误监控
- 上线 checklist

## 4. P0 / P1 / P2 优先级
### P0：必须做
- 工程骨架
- i18n 预埋
- 数据库与认证基础设施
- 首页
- 起卦问题页
- 摇卦交互页
- 排盘引擎
- 生成过渡页
- AI 分析服务
- 结果页
- 游客 1 次体验机制
- 注册后保存结果

### P1：强烈建议首版做
- 历史记录页
- tooltip 术语解释
- 示例问题优化
- 重新起卦
- 分享链接
- 分享图片
- 摇卦页体验打磨
- 结果页视觉打磨
- 埋点系统
- 基础后台报表

### P2：后续优化
- 首页高保真动态品牌打磨
- 更多分享策略
- 更丰富的结果摘要组织
- 分析质量回溯工具
- 用户反馈机制
- 轻内容模块
- 多语言真正上线

## 5. 核心数据与系统结构
### 确定性排盘层
输入：摇卦结果
输出：结构化 chart JSON

### skill 分析层
输入：标准化排盘 JSON
输出：
- 用神
- 旺衰
- 动变
- 象法
- 应期
- 结论

### 模型解释层
输出：
- 初步结论
- 白话分析
- 专业分析
- 分享摘要文案

## 6. 后台指标
必须可查看：
- 首页访问量
- 点击开始起卦数
- 进入摇卦页数
- 完成摇卦数
- 到达结果页数
- 点击注册数
- 完成注册数
- 保存卦例数
- 分享点击率
- 二次回访率
- 分析失败率
- 生成耗时

## 7. 开发时的文档沉淀规则
**后续所有规划必须同步留在项目空间中，不能只停留在聊天或飞书里。**

至少遵守：
- 产品规划 → `docs/PRD.md`
- 设计方向 → `docs/design-direction.md`
- 开发路线 / 优先级 → `docs/roadmap.md`
- 新的重要决策 → 追加到相应 docs，而不是只写在聊天里

## 8. 协作提醒
- 有阶段性成果，主动通知
- 有卡点，主动求助
- 不要等用户 repeatedly 追问进度
- 不要死循环空耗时间
