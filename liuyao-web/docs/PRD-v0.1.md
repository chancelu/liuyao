# 六爻在线网站 MVP PRD 草案 v0.1

## 1. 文档信息
- 项目名称：六爻在线占卦网站（暂定名）
- 版本：MVP v0.1
- 阶段：产品定义 / 方案确认
- 目标：完成从提问、摇卦、排盘到 AI 解读的在线闭环体验，并验证用户使用与注册保存链路

## 2. 产品概述

### 2.1 产品一句话
一个将传统六爻体验现代化、在线化、可视化的网页产品。用户可以在网站中完成提问、摇卦、排盘，并获得基于大模型与六爻 skill 的结构化分析结果。

### 2.2 产品定位
第一版不是纯硬核命理工具，也不是老派算命站，而是：
> 一个具有高级审美、神秘但克制、体验顺滑的在线占卦生活工具。

### 2.3 目标用户

**核心用户：**
- 小白用户：对六爻不懂或几乎不懂，对占卦/命理有兴趣，更关注"结果好不好懂""流程顺不顺"
- 兴趣用户：对星座、塔罗、玄学内容有消费习惯，愿意尝试新颖、有氛围的在线体验，重视审美与分享感
- 略懂爱好者：知道六爻是什么，对排盘、用神、应期等有一点兴趣，希望看到稍专业的分析内容

**非目标用户（MVP）：**
- 高度专业的六爻研究者
- 希望使用复杂起卦法的资深用户
- 需要完整学习体系的教学型用户
- 需要人工师傅深度互动服务的用户

## 3. 产品目标与成功标准

### 3.1 MVP 核心目标
完成并验证以下闭环：
1. 用户进入网站
2. 愿意开始起卦
3. 完成提问与摇卦
4. 获得排盘与解读结果
5. 愿意注册保存
6. 愿意再次回来使用或分享

### 3.2 MVP 不追求
- 商业变现闭环
- 人工服务闭环
- 高级命理功能全覆盖
- 多种复杂起卦法
- 社区内容生态

### 3.3 成功验证指标

**漏斗指标：**
- 首页访问量
- 点击"开始起卦"转化率
- 进入摇卦页人数
- 完成 6 次摇卦人数
- 成功到达结果页人数
- 结果页到注册页点击率
- 注册完成率
- 保存卦例率

**用户行为指标：**
- 结果页平均停留时长
- 分享点击率
- 分享完成率
- 二次起卦率
- 注册用户回访率

**产品质量指标：**
- 排盘生成成功率
- AI 分析成功率
- 平均分析时长
- 页面异常率 / API 失败率

## 4. 用户故事

### 4.1 游客用户故事
- **US-01**：作为一个第一次来网站的用户，我想先体验一次完整起卦流程，这样我能判断这个网站值不值得继续使用。
- **US-02**：作为一个不懂六爻的小白，我希望系统能引导我提问，并给我看得懂的结果，这样我不会被专业术语劝退。
- **US-03**：作为一个体验完结果的游客，我希望能在想保存结果时再注册，这样我不用一开始就被登录流程打断。

### 4.2 注册用户故事
- **US-04**：作为注册用户，我希望保存我的卦例和结果，这样以后可以回来回看。
- **US-05**：作为略懂六爻的用户，我希望除了白话解释，还能查看更专业的排盘与分析依据，这样我会觉得内容更可信、更有料。
- **US-06**：作为用户，我希望把结果分享给朋友或者发到社交平台，这样我可以传播这个体验或记录这次结果。

## 5. 核心产品原则

### 5.1 体验优先
第一版优先保证：页面气质、流程顺滑、结果可读、仪式感

### 5.2 排盘与分析分离
- 排盘引擎：确定性、规则化
- 分析引擎：大模型 + 六爻 skill
- 不能把排盘直接交给模型自由生成

### 5.3 双层结果
- 白话层：小白看得懂
- 专业层：爱好者觉得可信

### 5.4 先体验后注册
游客先完成一次体验，再引导注册保存。

### 5.5 高级模式暂不进入 MVP

## 6. 产品范围（MVP Scope）

### 6.1 In Scope

**核心功能：**
- 首页
- 游客体验一次起卦
- 提问表单
- 模拟摇卦（6 次）
- 自动排盘
- AI 分析
- 结果页
- 邮箱注册 / 登录
- 保存卦例
- 历史记录页

**MVP 中必须做进去的增强项：**
- 示例问题引导
- 术语解释 tooltip
- 重新起卦
- 一键分享链接
- 分享结果图

### 6.2 Out of Scope
- 高级起卦模式（时间起卦、报数起卦等）
- 多语言
- 支付 / 会员
- 社区 / 评论
- 用户补"后来应验了吗"
- 人工解盘
- 移动端 App（网页先行）

## 7. 功能需求

### 7.1 首页
- 展示产品主标题、副标题
- 提供明确 CTA：开始起卦
- 展示轻量使用流程说明（3 steps）
- 展示示例问题引导
- 提供轻量"什么是六爻"说明
- 支持游客直接进入起卦流程

### 7.2 起卦问题页
- 问题输入框（必填）
- 分类选择（感情 / 事业 / 财运 / 健康 / 学业 / 失物 / 其他）
- 时间范围选择（近期 / 本月 / 本年 / 未限定）
- 背景补充（选填）
- 示例问题引导，可点击填入
- 提交后进入摇卦页

### 7.3 摇卦页
- 展示当前问题摘要
- 展示当前第几爻 / 总进度
- 模拟三枚铜钱摇卦
- 每次摇卦结果自动记录，共 6 次
- 支持重新开始
- 完成后自动进入排盘生成页

### 7.4 排盘生成过渡页
- 展示主卦 / 变卦基础信息
- 展示分析进度文案
- 页面不应是纯 loading spinner
- 完成后自动跳转结果页

### 7.5 结果页
**页面展示顺序（已确认）：先排盘 → 再初步结论 → 再详细分析**

- 第一层：排盘总览（问题摘要、时间、本卦/变卦、动爻、简化排盘视图）
- 第二层：初步结论（一句话判断、当前趋势简述、短解释）
- 第三层：详细分析（白话版 + 专业版）
- 第四层：专业分析模块（用神、旺衰、动变、世应、六亲六神、象法、应期）
- Tooltip：专业术语支持点击/悬浮解释
- 操作区：保存结果、分享链接、生成分享图、重新起卦

### 7.6 注册 / 登录页
- 邮箱注册、邮箱登录
- 登录状态保持
- 游客第一次体验后引导注册
- 保存结果时若未登录，弹出注册引导

### 7.7 历史记录页
- 展示卦例列表（问题标题、日期、分类、一句话结论、本卦/变卦摘要）
- 点击进入详情页
- 支持删除（可选）、再次起卦

### 7.8 分享功能
- 分享链接：可复制当前结果页链接，可生成对外分享页
- 分享图片：生成高审美结果图（问题摘要、日期、本卦/变卦、一句话结论、品牌标识）

## 8. AI 与分析系统需求

### 8.1 系统架构原则
- 层 1：排盘引擎（规则层）— 确定性，不依赖模型
- 层 2：六爻分析引擎（skill 层）— 接收标准化排盘 JSON，调用六爻 skill
- 层 3：解释输出层（模型层）— 将 skill 结果组织成白话版和专业版输出

### 8.2 输出要求
系统同一次分析需至少产出：
- 初步结论
- 白话分析
- 专业分析
- 结构化分析 JSON（后台可用）
- 分享摘要文案

## 9. 已确定的关键决策
- 注册方式：先邮箱注册
- 高级模式：第一版不做
- 目标用户：小白 + 兴趣用户 + 略懂爱好者
- 产品气质：体验优先、界面 fancy、生活化
- 商业模式：第一版不收费，先验证
- 结果展示：兼顾大白话 + 专业术语解释
- 注册策略：先体验，后注册保存
- 首页语气：混合（神秘高级氛围 + 小白友好 + 专业深度）
- 结果页第一屏：先排盘 → 再初步结论 → 再详细分析
- 游客体验次数：游客 1 次，注册后更多
- 分享：允许一键分享（链接 + 图片）
- 首页内容感：带一点，但很轻
