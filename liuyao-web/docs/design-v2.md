# Yarrow 爻镜 — 设计重构文档 v1.0

> "让用户忘记自己在用网站。"

---

## 一、品牌

### 名称
- 中文：爻镜
- 英文：Yarrow（蓍草，传统易经占卜工具）
- 标语（中）：六爻在线占卦
- 标语（英）：Ancient Chinese divination, reimagined.

### 品牌视觉锚点
- 一个大字"爻"作为整个网站的视觉核心
- 不需要 logo 图形，字即 logo
- "Yarrow" 用衬线体，letter-spacing 0.3em，与"爻镜"并列时形成中西对照

## 二、设计哲学

### 核心理念
每一个页面都是一次仪式空间的进入。不是展示信息，而是营造氛围。
不是给用户看一个漂亮的网站，而是让用户忘记自己在用网站。

### 三条设计原则

1. **空即是满** — 留白不是空，是气。日本设计里的"間"（ma）。空间本身承载意义。
2. **克制即力量** — 每个元素都必须有存在的理由。去掉一切装饰性的、不承载意义的东西。
3. **仪式感优先** — 用户不是在"使用工具"，而是在"进行一次占卜仪式"。每个交互都应该有重量。

### 设计路线
以【东方神秘主义】为主体，融入【极简克制】和【现代宇宙感】：

- **东方神秘主义（主体）**：水墨纹理、古铜质感、传统纹样现代化、竖排文字点缀
- **极简克制（融入）**：大量留白、内容即设计、高对比度、去掉不必要的装饰
- **现代宇宙感（融入）**：适度金色光晕、克制的玻璃拟态、流畅过渡、现代间距系统

## 三、色彩系统

```
背景层级：
  --bg-deep:       #0D0B08    （带暖调的近黑，像夜色中的宣纸）
  --bg-mid:        #11100C
  --bg-elevated:   #16140F
  --bg-card:       rgba(196, 149, 107, 0.02)

文字层级：
  --text-primary:  #F0EBE3    （暖白，不是纯白）
  --text-secondary:#D4CFC6
  --text-muted:    #8A8070    （暖灰）
  --text-dim:      #5A5348    （深暖灰）

主色：
  --gold:          #C4956B    （红铜金）
  --gold-light:    #D4A87B
  --gold-dim:      rgba(196, 149, 107, 0.30)

点缀色：
  --cinnabar:      #8B3A3A    （朱砂，极少量使用）

边框/分隔：
  --border:        rgba(196, 149, 107, 0.06)
  --border-hover:  rgba(196, 149, 107, 0.12)

功能色：
  --success:       #7AB88A
  --error:         #C47070
```

### 时间感知色温
网站根据当前时辰微调背景色温（偏移 2-3%，用户无意识感知）：
- 子时（23-1）：偏冷，bg #0B0A0D
- 卯时（5-7）：微暖，bg #0D0B08
- 午时（11-13）：最暖，bg #0F0C08
- 酉时（17-19）：偏铜，bg #0E0B07
- 实现方式：CSS custom property + JS 定时更新

## 四、字体与排版

### 字体
- 主字体：Noto Serif SC（保持）
- 英文辅助：系统衬线体 Georgia fallback
- 品牌名"Yarrow"：考虑后续引入一款西文衬线体（如 Cormorant Garamond）

### 排版规范
- 正文：15px，line-height 2.0（松散，有呼吸感）
- 标题：极细体（font-weight 200），letter-spacing 0.08-0.12em
- 品牌名：letter-spacing 0.3-0.5em
- 标签/小字：10px，tracking 0.4em，uppercase
- 关键位置竖排文字：writing-mode: vertical-rl，作为氛围装饰

### 文案声音
整个网站的文案像一个沉稳的长者在说话，不是 UI 在提示：
- "请输入问题" → "心中所问"
- "提交" → "问卦"
- "开始摇卦" → "掷卦"
- "加载中" → 不说话，只有氛围
- "登录" → "入"
- "登出" → "离"
- "历史记录" → "往卦"
- "个人中心" → "我的"
- "积分" → "功德"（待定，或保持"积分"）

## 五、页面设计

### 5.1 首页 — "暗室一灯"

**现状问题**：信息过多（Hero + 预览卡片 + 步骤 + FAQ），像产品介绍页。

**重构**：
- 进入首页，几乎是空的
- 中央：一个大字"爻"（font-size 约 120-160px），下方小字"Yarrow · 六爻在线占卦"
- 唯一交互：一个"问卦"按钮，或直接点击"爻"字进入
- 背景：极淡水墨晕染（CSS radial-gradient），随呼吸节奏微微明暗（3s 周期，opacity 0.02-0.04）
- 导航栏极简：左"爻 Yarrow"，右只有用户入口
- 去掉预览卡片、步骤说明
- FAQ 移到页脚折叠或独立页面
- 首次访问的西方用户看到英文副标题："Ancient Chinese divination, reimagined."

**视觉层次**：
```
[极淡水墨背景 — 呼吸明灭]
         
         爻
   Yarrow · 六爻在线占卦

       [ 问 卦 ]
```

### 5.2 起卦页 — "静心"

**现状问题**：表单感太强，像填问卷。

**重构**：
- 页面中央只有问题输入区域，大面积留白
- 输入框：无边框，只有底部一条细线（金色，极淡），像在宣纸上书写
- 分类选择：默认折叠，点击"选择类别"展开，选完自动收起
- 时间范围：同上，折叠式
- 背景补充（选填）：折叠在"补充背景"链接下
- 提交按钮："问卦"两个字，居中，圆形或椭圆
- 整体感觉：安静、专注、只有你和你的问题

**视觉层次**：
```
         心中所问
  ___________________________
  |                           |
  |   （用户输入问题）          |
  |___________________________|

     感情 · 近期          ← 小字，可点击修改

         [ 问 卦 ]
```

### 5.3 摇卦仪式页 — "核心体验"

**现状问题**：铜钱动画 + 进度显示，功能正确但仪式感不够。

**重构**：
- 整个屏幕只有三枚铜钱和虚空
- 去掉所有 UI 元素（进度条、爻名标签、"第X爻"提示）
- 点击/触摸屏幕任意位置即可摇卦（不是点按钮）
- 铜钱动画 + 声音保持
- 每一爻生成后，屏幕下方缓缓浮现一条线：
  - 阳爻：一条完整的横线
  - 阴爻：中间断开的两段线
  - 动爻：线条带微微金色光晕
- 线条出现的动画像毛笔落笔（从左到右渐现，800ms）
- 六爻完成后，停顿 1.5 秒，卦象整体微微发光，然后自动跳转
- 背景随进度微变：从纯黑 → 第六爻时极淡金色晕染
- 唯一的文字提示：第一爻前，屏幕中央淡淡显示"触摸掷卦"，掷完第一次后消失

**视觉层次**：
```
[第1爻时]

         🪙 🪙 🪙
         
         ———————          ← 刚生成的第一爻

[第6爻完成时]

         🪙 🪙 🪙

         ——— ———          上爻
         ———————          五爻
         ——— ———          四爻
         ———————          三爻
         ———————          二爻
         ———————          初爻
```

### 5.4 结果页 — "展卷"

**现状问题**：信息密度高，像报告。

**重构**：
- 顶部：卦象居中，用粗细线条绘制（阳爻粗线，阴爻两段细线），像书法
- 卦名大字居中，下方宫位、五行小字
- 如有变卦，本卦 → 变卦 用一个极淡的箭头或"→"连接
- AI 解读分段展开：
  - 用户向下滚动，内容像卷轴展开
  - 每个段落之间用传统分隔符（◇ 或细线 + 小符号）
  - 段落进入视口时 fade-in（不是 slide-up）
- 配色：正文暖白，关键词/卦名金色，整体克制
- 底部：分享按钮 + 返回首页，极简

### 5.5 历史记录页 — "往卦"

- 列表式，每条记录是一张极简卡片
- 显示：日期、问题（截断）、卦名
- 卡片左侧用一个小的卦象线条图标
- 空状态："尚无往卦" + 一个"问卦"入口

### 5.6 登录页

- 保持现有双 tab（验证码 + 密码）结构
- 样式跟随新设计系统
- 输入框：底部细线风格
- 页面顶部一个"爻"字作为视觉锚点

### 5.7 个人中心

- 头像 + UID + 邮箱 + 积分，竖向排列，大量留白
- 积分显示用金色
- 操作项（换绑邮箱等）用文字链接，不用按钮

### 5.8 管理后台

- 保持功能性优先，跟随新色彩系统即可
- 不需要特殊设计

### 5.9 分享页 & 分享图

- 分享图重新设计：像一张可收藏的艺术品
- 包含：卦象线条画（居中）、卦名、一句核心解读、"Yarrow 爻镜"品牌
- 配色：深色底 + 金色线条 + 暖白文字
- 比例：适合社交媒体（1:1 或 4:5）

## 六、交互与动效

### 动效原则
- 所有过渡 600-800ms，缓动 ease-out
- 页面进入：整体 fade-in（不要 translateY）
- 唯一的"动"效果：呼吸感明暗变化（opacity 微调）
- 铜钱动画保持现有方案
- hover 效果：极淡金色光晕扩散，不要位移

### 声音设计
- 铜钱碰撞：保持现有方案
- 其他页面：静默（不加背景音乐或提示音）
- 未来可考虑：摇卦完成时一声极淡的铜磬

### 等待状态 = 冥想
- AI 分析生成时，不显示进度条
- 显示一句古语或卦辞，缓缓呼吸明灭（3s 周期）
- 例句轮换："天行健，君子以自强不息"、"地势坤，君子以厚德载物"
- 让等待本身成为仪式的一部分

### 错误状态的人格
- 404："此卦无解"
- 网络错误："天地未通，稍后再试"
- 服务异常："卦象暂隐，请稍候"
- 保持深色背景 + 金色文字，不破坏氛围

## 七、响应式策略

### 移动端
- 首页"爻"字缩小到 80px
- 摇卦页：全屏体验，隐藏导航栏
- 结果页：单列，段落间距加大
- 触摸反馈：摇卦时考虑 navigator.vibrate() 轻触觉

### 桌面端
- 最大内容宽度 720px（比现在窄，更聚焦）
- 首页"爻"字可以到 160px
- 两侧留白区域可放竖排装饰文字

## 八、国际化准备

### 双语支持
- 品牌名始终显示 "Yarrow 爻镜" 或 "爻 Yarrow"
- 首页副标题英文："Ancient Chinese divination, reimagined."
- 未来：完整英文版本，所有文案翻译
- 卦名保留中文 + 拼音 + 英文释义（如 "乾 Qián — The Creative"）

### 文化桥梁
- 对不了解六爻的用户，用一句话解释而非教程
- 强调体验而非知识门槛
- 视觉语言本身就是跨文化的（线条、留白、光影）

---

## 九、技术约束

- 只改样式和布局，不改业务逻辑
- 不引入新的 npm 依赖
- 所有装饰用 CSS/SVG 实现，不用外部图片
- 移动端响应式必须良好
- 改完确保 `npm run build` 通过
- 时间感知色温用 JS + CSS custom property 实现

## 十、实施优先级

### Phase 1 — 设计系统（globals.css）
建立新的色彩、字体、卡片、按钮、动效基础

### Phase 2 — 核心体验（首页 + 摇卦 + 结果页）
这三个页面决定了用户对产品的第一印象

### Phase 3 — 辅助页面（起卦、历史、登录、个人中心）
跟随新设计系统调整

### Phase 4 — 细节打磨（错误状态、等待状态、分享图、时间感知）
提升整体完成度
