# 六爻网站 MVP PRD

## 1. 产品概述
- 项目定位：一个将传统六爻体验现代化、在线化、可视化的网页产品。
- 第一版不是硬核命理工具，也不是老派算命站，而是一个**高级审美、神秘但克制、体验顺滑**的在线占卦生活工具。
- 目标：完成从提问、摇卦、排盘到 AI 解读的在线闭环，并验证用户使用、注册保存、回访与分享链路。

## 2. 目标用户
### 核心用户
- 小白用户：不懂或几乎不懂六爻，更关注结果是否好懂、流程是否顺滑。
- 兴趣用户：对占卦、星座、塔罗、玄学内容有消费习惯，重视体验、审美和分享感。
- 略懂爱好者：对六爻有一点了解，希望看到排盘、术语与分析依据。

### 非目标用户（MVP）
- 高度专业的六爻研究者
- 需要复杂起卦法的资深用户
- 需要完整教学体系的用户
- 需要人工师傅深度互动服务的用户

## 3. 已确认的关键决策
- 注册方式：**邮箱注册 / 登录**
- 目标用户：**小白 + 兴趣用户 + 略懂爱好者**
- 产品气质：**fancy、体验优先、现代东方感、生活化，不是工具站**
- 商业模式：**第一版不收费，先验证产品通路**
- 展示策略：**兼顾大白话 + 专业术语解释**
- 注册策略：**先体验，后注册保存**
- 游客额度：**游客 1 次，注册后更多**
- 语言策略：**第一版中文优先，但技术架构预埋 i18n**

## 4. 产品主张
几分钟完成提问、摇卦、排盘、解读的在线六爻体验。

更偏用户感知的版本：
> 把传统六爻变成一套现代、好看、易懂的在线占卦体验。

## 5. MVP 核心目标
验证以下闭环：
1. 用户进入网站
2. 愿意开始起卦
3. 完成提问与摇卦
4. 获得排盘与解读结果
5. 愿意注册保存
6. 愿意再次回来使用或分享

## 6. MVP 范围
### In Scope
- 首页
- 游客体验一次起卦
- 提问表单
- 模拟摇卦（6 次）
- 自动排盘
- AI 分析
- 结果页
- 邮箱注册 / 登录
- 保存卦例
- 历史记录页
- 示例问题引导
- 术语解释 tooltip
- 重新起卦
- 一键分享链接
- 分享结果图

### Out of Scope
- 高级起卦模式（时间起卦、报数起卦等）
- 多语言正式上线
- 支付 / 会员
- 社区 / 评论
- 用户补“后来应验了吗”
- 人工解盘
- App 端

## 7. 用户主流程
### 游客路径
首页 → 起卦问题页 → 摇卦页 → 排盘生成页 → 结果页 → 注册保存

### 注册用户路径
首页 / 历史页 → 起卦问题页 → 摇卦页 → 排盘生成页 → 结果页 → 保存 / 分享 / 回看

## 8. 页面与功能要求
### 8.1 首页
目标：建立品牌气质、解释产品价值、促成开始起卦。

必须包含：
- Hero 区
- 三步说明
- 示例问题引导
- 轻量“什么是六爻”说明
- 主 CTA：开始起卦

### 8.2 起卦问题页
字段：
- 问题输入（必填）
- 分类：感情 / 事业 / 财运 / 健康 / 学业 / 失物 / 其他
- 时间范围：近期 / 本月 / 本年 / 未限定
- 背景补充（选填）
- 示例问题引导

原则：帮助用户形成一个可问的问题，不要让小白卡住。

### 8.3 摇卦页
要求：
- 连续 6 次摇卦
- 显示当前第几爻 / 总进度
- 每次有清晰反馈
- 支持重新开始
- 完成后自动进入排盘生成页

原则：做出仪式感，不要像普通表单工具。

### 8.4 排盘生成过渡页
要求：
- 展示主卦 / 变卦基础信息
- 展示分析阶段文案，如：正在装盘、正在推演用神与旺衰、正在生成结果
- 不使用纯 loading spinner 风格

### 8.5 结果页
**已确认顺序：先排盘，再初步结论，再具体分析。**

第一层：排盘总览
- 问题摘要
- 时间信息
- 本卦 / 变卦
- 动爻
- 简化排盘视图
- 排盘必须具备审美感，不能像纯工具表格

第二层：初步结论
- 一句话判断
- 当前趋势简述
- 简短解释

第三层：详细分析
- 白话版（默认展开）
- 专业版（可折叠）
  - 用神
  - 旺衰
  - 动变
  - 世应
  - 六亲六神
  - 象法
  - 应期

第四层：术语解释
- tooltip 解释用神、世应、旬空、六冲、六合等术语

第五层：操作区
- 保存结果
- 分享链接
- 生成分享图
- 重新起卦

### 8.6 注册 / 登录页
要求：
- 邮箱注册
- 邮箱登录
- 保持登录态
- 游客体验后引导注册
- 保存结果时未登录则弹出注册引导

### 8.7 历史记录页
每条记录展示：
- 问题标题
- 日期
- 分类
- 一句话结论
- 本卦 / 变卦摘要
- 进入详情
- 再次起卦

### 8.8 分享功能
- 分享链接
- 分享页
- 分享图片
- 分享图需为品牌化结果卡片，不是网页截图

## 9. 结果展示策略
第一屏不要直接扔一大堆术语。

建议层级：
1. 排盘（建立可信度）
2. 初步结论（让用户先知道结果）
3. 白话分析（小白可读）
4. 专业分析（爱好者可展开）
5. tooltip 解释术语

## 10. 技术与系统原则
### 三层分离
1. **排盘引擎（确定性层）**
   - 摇卦结果转爻
   - 主卦 / 变卦
   - 动爻
   - 世应
   - 纳甲
   - 六亲
   - 六神
   - 月建 / 日辰 / 旬空

2. **六爻分析引擎（skill 层）**
   - 使用 `liuyao-calculation`
   - 进行用神、旺衰、动变、象法、应期分析

3. **解释输出层（模型层）**
   - 生成白话版
   - 生成专业版
   - 统一语气与结构化输出

原则：**排盘与分析严格分开，不能把排盘交给模型自由生成。**

## 11. 成功验证指标
这些指标需在后台系统中可视化：

### 漏斗指标
- 首页访问量
- 点击开始起卦转化率
- 进入摇卦页人数
- 完成 6 次摇卦人数
- 成功到达结果页人数
- 结果页到注册页点击率
- 注册完成率
- 保存卦例率

### 用户行为指标
- 结果页平均停留时长
- 分享点击率
- 分享完成率
- 二次起卦率
- 注册用户回访率

### 产品质量指标
- 排盘生成成功率
- AI 分析成功率
- 平均分析时长
- 页面异常率 / API 失败率

## 12. 风险与注意事项
- 排盘错误会毁全局：排盘层必须可测试、可验证。
- AI 结果可能不稳定：必须固定结构化输入输出，先规则后解释。
- 小白看不懂：默认展示白话版，专业术语必须可解释。
- 视觉跑偏会像工具站或旧站：首页、摇卦页、结果页需重点打磨。
